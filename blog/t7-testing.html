Variable scope

What 'scope' is, and how it works in JavaScript.

The term “variable scope” refers to the context of the variable and whether it is available for use by different parts of the program.

It is controlled by where it is declared in the code and this determines if it is accessible or not by that part of the program.

Javascript has two scopes.

Local variables are ones which are declared inside a function and are only accessible by that function or other functions nested within in.

Global variables are declared outside the function and are visible and accessible from any part of the program.

var name = 'Mike'; // global variable - declared outside the function and visible everywhere

        function showName() {
          var name = 'Ximing'; // local variable - declared inside the function showName
          console.log(name); // returns Ximing
        }
        console.log(name); // returns Mike

        showName(); // returns Ximing

If variables within the code have the same name, the local one within the function will take precedence and will be used first. It is good practice to declare local variables before you use them.
