HTML, CSS and DOM 1 (Part 1)

The terms HTML, CSS and DOM are important in web development. What do they mean?

Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

HTML , CSS and the DOM are terms commonly used in the web development world to describe how a webpage is put together. Each part has it’s own function and performs a specific role in creating what you see on screen.

Using a tree as an analogy. When you look at a tree, think about the elements which you define it as what it is.

The trunk is what gives the tree structure. You could call this the DOM (Document object model) of the tree. This defines what the tree is going to look like in terms of its overall structure . The DOM is like a map or a model of how the web page is put together.

HTML (Hypertext Mark-up Language) is what gives the web page its form. It is comprised of various elements, each one with a specific function or role. Each element is identified by a tag showing its purpose.

Common examples of these tags include sections, headings, paragraphs, links and images. These tags are like the different parts of the tree - leaves, fuits, flowers, branches, roots.

The HTML allows us to put the elements onto our DOM structure much like how we would expect our tree to look.

Now what makes the tree different from all the other trees in the forest is how it appears. Each part of the tree can have it’s own shape, colour, smell, size. The combinations are endless.

This is what the CSS (cascading style sheet) does. It provides styling to the website and enables us to apply all different styles such as colour, formatting, font characteristics, shape.

This is what makes web pages beautiful.

What is meant by boxifying design?

To make the design process easy to create and understand, the page is split up into boxes and containers. This is called boxifying.

Each box holds elements of a certain type - headers, sidebars, footers, paragraphs, images etc and there can be boxes within boxes too.

By boxifying our design elements and showing our layout in a graphical sense, it makes the job of creating out HTML and CSS coding much easier.

What is the box model?

The box model defines how a box and its attributes relate to each other and how this box (and its contents may appear on the screen. There are 4 parts to the box model.

Content
Where the element resides.
Padding
The white space inside between the element and the box.
Border
Outlines the visible portion of the element.
Margin
The invisible space outside of the box.

The actual size of the box includes all these attributes so it is important to consider this when laying out the page. For example, if an element has been given a width attribute of 100px, the true size of the box would have to include the left/right padding, border-width and margin dimensions.

One way of reducing unintended formatting errors would be to use the box-sizing selector using the border-box attribute to limit the size of the box to the border itself.
